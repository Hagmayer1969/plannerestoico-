# Task 1.0 — Gerenciar estado e persistência de dados do ritual 3-1-3 e do planner

- **Complexidade:** HIGH
- **Dependências:** 0
- **Status:** completed

## Objetivo
Transformar a interface existente em um fluxo de dados funcional, garantindo que o usuário possa digitar os campos do ritual 3-1-3 e visualizar os compromissos do dia com estado persistente localmente.

## Contexto PRD
O PRD exige um ritual 3-1-3 e um planner diário funcional, com a experiência fluindo entre onboarding, intro e dashboard. Atualmente, o dashboard renderiza campos de entrada estáticos sem salvar dados.

## Requisitos Técnicos
- Criar estado para os campos de gratidão (3 itens), alvo do dia e pontos de melhoria (3 itens).
- Criar estado para compromissos do dia com título, horário e conclusão.
- Salvar e recuperar dados usando `localStorage`.
- Garantir que o estado seja restaurado ao recarregar a página.

## Critérios de Aceitação
- O usuário pode preencher os campos do ritual 3-1-3 e eles permanecem no estado durante a sessão.
- Os dados do ritual são salvos em `localStorage` e reaparecem após recarregar o navegador.
- O planner de compromissos exibe items a partir do estado e salva as alterações localmente.
- Não há regressão no layout existente.

## Subtarefas
1. Auditar `src/components/Dashboard.tsx` e listar os elementos interativos atuais.
2. Adicionar estados React para os campos de gratidão, objetivo do dia e melhorias.
3. Adicionar estado React para compromissos de exemplo.
4. Persistir e recuperar os dados com `localStorage`.
5. Verificar o comportamento no fluxo atual sem alterar layout.

## Testes Planejados
- Teste de unidade: função utilitária de `localStorage` (serialização/deserialização).
- Teste de integração simples: simular carregamento do dashboard e garantir restauração do estado.
- Teste manual: recarregar o app e verificar que os campos preenchidos reaparecem.

## Resultado
- Implementado estado persistente para ritual 3-1-3 e compromissos no `Dashboard`.
- Adicionado utilitário de `localStorage` em `src/lib/localStorage.ts`.
- Código validado com `npm run lint`.
