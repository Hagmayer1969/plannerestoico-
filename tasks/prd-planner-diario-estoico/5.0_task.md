# Task 5.0 — Criar testes de unidade e integração para componentes principais

- **Complexidade:** MEDIUM
- **Dependências:** 1.0
- **Status:** completed

## Objetivo
Adicionar cobertura de testes para os componentes principais do app e validar o fluxo básico de onboarding → intro → dashboard.

## Contexto PRD
Os requisitos do PRD e da spec pedem uma experiência confiável. Testes ajudam a garantir que as mudanças de estado e UI continuem funcionando conforme esperado.

## Requisitos Técnicos
- Definir um framework de testes adequado para o projeto.
- Criar testes para: Dashboard, Onboarding, e utilitários de persistência.
- Verificar o fluxo de navegação principal.

## Critérios de Aceitação
- Há ao menos um teste automatizado cobrindo persistência de dados.
- Há ao menos um teste de renderização para Dashboard ou Onboarding.
- Os testes podem ser executados com um comando claro no projeto.

## Subtarefas
1. Escolher e configurar o framework de testes (Vitest ou Jest).
2. Criar testes para utilitários de `localStorage`.
3. Criar testes de renderização para Dashboard.
4. Documentar o comando de execução de testes.

## Testes Planejados
- Teste unitário: utilitário de leitura/escrita em `localStorage`.
- Teste de componente: renderização do `Dashboard` com campos básicos.
- Teste de fluxo: navegação entre onboarding e dashboard se possível.

## Resultado
- Configurado `vitest` no projeto e adicionada a linha `test` em `package.json`.
- Criados testes em `src/lib/localStorage.test.ts`, `src/lib/stoicThemes.test.ts` e `src/components/Onboarding.test.tsx`.
- Todos os testes passaram com `npm run test -- --run`.
- O projeto foi validado com `npm run build`.
