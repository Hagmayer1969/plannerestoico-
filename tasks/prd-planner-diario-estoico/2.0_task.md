# Task 2.0 — Implementar planner de compromissos com adição e marcação

- **Complexidade:** MEDIUM
- **Dependências:** 1.0
- **Status:** completed

## Objetivo
Permitir que o usuário adicione novos compromissos, visualize a lista diária e marque itens como concluídos, mantendo a experiência consistente com o layout atual.

## Contexto PRD
O PRD define um planner de compromissos integrado ao dashboard como parte da rotina diária. O design atual apresenta uma lista de compromissos e botão para adicionar novos itens.

## Requisitos Técnicos
- Adicionar um formulário/disparador para criar novos compromissos.
- Salvar compromissos com título, horário e status de conclusão.
- Atualizar a lista de compromissos no UI sem quebrar o layout.
- Persistir alterações em `localStorage`.

## Critérios de Aceitação
- O usuário pode adicionar um novo compromisso via interface.
- O usuário pode marcar um compromisso como concluído.
- A lista de compromissos permanece após recarregar a página.
- A experiência visual segue o mesmo layout atual.

## Subtarefas
1. Criar campos de entrada e botão de adição no componente de agenda.
2. Mapear os compromissos do estado para a lista apresentada.
3. Adicionar lógica para alternar o status de completude.
4. Atualizar persistência local.

## Testes Planejados
- Teste de unidade: gerenciar lista de compromissos (adicionar, alternar status).
- Teste de integração: fluxos de adicionar compromisso e recarregar o app.

## Resultado
- Implementado formulário inline de adição de compromisso.
- O botão "ADICIONAR COMPROMISSO" abre o formulário e permite salvar novo compromisso.
- Compromissos agora são marcáveis como concluídos e persistem em `localStorage`.
- Código validado com `npm run lint` e `npm run build`.
