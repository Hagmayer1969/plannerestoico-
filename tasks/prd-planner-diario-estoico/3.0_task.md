# Task 3.0 — Criar motor de lembretes estoicos e interface de ativação

- **Complexidade:** MEDIUM
- **Dependências:** 1.0
- **Status:** completed

## Objetivo
Implementar a interface de ativação de lembretes estoicos no dashboard e criar a base para o motor de notificações/alertas futuros.

## Contexto PRD
O PRD pede um motor de lembretes estoicos com um botão de ativação no dashboard. A versão inicial deve fornecer o acionamento visual e preparar a estrutura para notificações reais.

## Requisitos Técnicos
- Adicionar estado para armazenar se o lembrete está ativado.
- Atualizar o botão de ativação para refletir o estado atual.
- Salvar preferência em `localStorage`.
- Criar um placeholder de lógica que possa ser estendido para notificações reais.

## Critérios de Aceitação
- O usuário pode ativar/desativar os lembretes pelo dashboard.
- O estado de ativação é persistido entre recargas.
- O botão visual permanece no mesmo layout atual.

## Subtarefas
1. Adicionar estado de lembretes no dashboard.
2. Atualizar o botão de ativação para alternar entre ativo/inativo.
3. Persistir a preferência em `localStorage`.
4. Garantir o layout intacto.

## Testes Planejados
- Teste de unidade: função de persistência do estado de lembretes.
- Teste de integração: alternar lembretes e recarregar o app.

## Resultado
- Implementado toggle de lembretes estoicos no dashboard.
- O estado de ativação é persistido em `localStorage`.
- O botão de ativação mantém o mesmo layout visual e usa o texto `ATIVAR` / `ATIVADOS`.
- Código validado com `npm run lint` e `npm run build`.
